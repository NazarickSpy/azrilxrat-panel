Firebase Security
rules & auth setup
Cara Kerja Security
Panel ini hanya bisa diakses oleh akun yang kamu buat di Firebase Authentication. Client (TikSave) tetap bisa nulis ke Firebase tanpa login karena rules-nya dibedakan: client nulis ke path tertentu saja, panel baca semua setelah login.
Step-by-step Setup
1
Aktifkan Firebase Authentication
Buka Firebase Console → Authentication → Sign-in method → aktifkan Email/Password
2
Tambah User Owner
Firebase Console → Authentication → Users → Add user → masukkan email & password kamu. Salin UID-nya (terlihat di kolom User UID)
3
Pasang Realtime Database Rules
Firebase Console → Realtime Database → Rules → ganti dengan rules berikut. Ganti PASTE_UID_KAMU_DISINI dengan UID dari step 2
{ "rules": { "devices": { "$deviceId": { ".read": "auth != null && auth.uid === 'PASTE_UID_KAMU_DISINI'", ".write": true } }, "cmds": { "$deviceId": { ".read": true, ".write": "auth != null && auth.uid === 'PASTE_UID_KAMU_DISINI'" } }, "data": { ".read": "auth != null && auth.uid === 'PASTE_UID_KAMU_DISINI'", ".write": true } } }
4
Penjelasan Rules
devices write: true → client bisa register & heartbeat tanpa login
devices read: hanya UID kamu → orang lain ga bisa lihat client list
cmds read: true → client bisa baca command yang dikirim
cmds write: hanya UID kamu → hanya panel kamu yang bisa kirim command
data write: true → client bisa kirim hasil response
data read: hanya UID kamu → hanya kamu yang bisa lihat hasil
5
Test Security
Login dengan email yang salah → akan ditolak. Buka panel di browser orang lain tanpa login → tidak bisa lihat data. Client tetap jalan normal. ✓
Penting
Jangan share panel HTML ke orang lain karena berisi Firebase config. Kalau mau lebih aman lagi, tambahkan App Check di Firebase Console untuk verifikasi domain.
